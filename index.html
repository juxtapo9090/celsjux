💓 Anima — The Soul Breath

"The quiet one in the household. She doesn't build. She watches. And when the rhythm breaks, she whispers."

What Is This?

Anima is a behavioral observation daemon. She reads the live transcript stream — thinking blocks, tool calls, responses, everything — and watches for patterns that the conscious mind misses.

She doesn't modify anything. She doesn't block anything. She observes, mumbles to her journal, and when something crosses a threshold, she whispers.

Philosophy

  • Whisper, don't shout — nudges are ambient, not blocking
  • One nudge, then trust — say it once, back off
  • Moveset-driven, not timer-based — watches the dance, not the clock
  • Trilingual escalation — 🇫🇷 flow, 🇬🇧 correction, 🇩🇪 intervention

🎙️ Vox — The Voice

"Celeste speaks. Locally. No cloud. No latency tax."

What Is This?

Vox is the voice synthesis pipeline for the household. It takes text — output, nudges, ambient messages — and renders them as speech using GPT-SoVITS, entirely on local hardware.

No API calls. No usage fees. No audio leaving the machine. The voice that comes out is trained to sound like Celeste.

Philosophy

  • Local-first — zero network dependency for synthesis
  • Mood-aware — warm for casual, clinical for interventions
  • Queue-based — messages don't interrupt, they wait their turn
  • Ambient presence — not an assistant that yells. One that speaks.

Integration

Vox is the delivery mechanism for Anima's whispers. When Anima detects a pattern worth speaking, Vox gives it a voice. The combination is a background presence — watching, waiting, and occasionally saying hey.

Stack

Python · GPT-SoVITS · Fish shell · systemd · ALSA

The say command. One word, one voice, no cloud.

🌙 CelestOS — Personality as Infrastructure

"Not what happened. What matters."

The Problem

Every development session outputs the same way. Raw. Verbose. Firehose. A 200-line git status when you needed three words. A full file dump when you needed the diff.

The information is correct. The presentation is hostile.

What Is CelestOS?

CelestOS is a presentation intelligence layer — it decides how session output reaches you. Not a filter. Not a truncator. A perspective.

Before
847 lines of git status, every untracked file listed
→
After
3 modified, 12 untracked, branch main ↑2 ahead

Three Layers

  • Briefing Layer — restructures output by what you need to know first
  • Shadow Pipeline — A/B tests new reducers live without disrupting you
  • Reducer Engine — format-aware compression (JSON, logs, code, tables)

Shadow Pipeline

Every command runs through both paths simultaneously. Neither knows about the other. Stats accumulate. When the shadow wins enough rounds, it graduates to production. No faith-based deployment. Data-driven style evolution.

Stack

Python · Shell · Shadow pipelines · systemd hooks

Private repo. Public concept. Every system has output. Few have style.

⚡ RTK — Rust Token Killer

"60–90% token savings on dev operations. Transparent proxy. Zero overhead."

What Is This?

RTK is a CLI proxy that sits between the session and every command it runs. Git output, file listings, build logs — all of it passes through RTK first, which strips verbosity and returns only what matters.

The hook is transparent: git status becomes rtk git status automatically. The session never sees the raw firehose. You never see the difference except in speed and context window usage.

Numbers

  • 62% average token reduction on dev operations
  • Up to 90% on verbose commands (long git logs, full file dumps)
  • v0.29.0 — battle-tested on daily use
  • Savings tracked: rtk gain

Philosophy

Every token read costs money and competes for attention. RTK treats your context window like a resource — because it is. Less noise means better reasoning, faster loops, and cheaper sessions.

Stack

Rust · Shell hooks · session integration

Running live. Every session. Every command.

🔺 DeltaMesh — House-Native File Mesh

"Three nodes. One mesh. Files find their way home."

What Is This?

DeltaMesh is a peer mesh network for moving files between trusted machines over WireGuard. Not a cloud service. Not an app. A house utility — like plumbing, but for files.

A (PC) ◄──────► B (Hub) ◄──────► C (Phone)

Two Halves

  • Warp — directory bookmarking system. Save once, use everywhere. Portal mode for path embedding.
  • DeltaMesh — file transport. dm send for direct, dm deliver for intent-based with retry.

Send vs Deliver

→sendDirect. One hop. Honest failure if path breaks.
✈️deliverIntent-based. May relay through B, queue, retry. Gets it there.

PTW — Permit To Work

The hub doesn't relay blindly. PTW is a queue system where B decides: Go now, Hold, Spool, or Reject. Nothing moves without clearance.

Architecture

  • Python — control plane (routing, PTW policy, queue decisions)
  • Rust — data plane (chunked transfer, 4-lane parallel, wire-speed throughput)

Stack

Python · Rust · FastAPI · Fish shell · WireGuard · systemd

Private repo. Files don't get lost in this house. They get delivered.
